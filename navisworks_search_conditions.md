# Типы условий поиска в Navisworks

## Реальные операторы сравнения в Navisworks

### Для строковых свойств (например, "Имя")
- **`=`** (Равно) - точное совпадение строки
- **`Не равно`** - не равно (обратное равно)
- **`Содержит`** - содержит подстроку
- **`Подстановочный знак`** - с использованием символов подстановки (*, ?)
- **`Определенный`** - имеет значение (не пустое)
- **`Не определено`** - не имеет значения (пустое)

### Для числовых свойств
- **`=`** (Равно) - точное совпадение
- **`Не равно`** - не равно
- **`>`** - больше чем
- **`<`** - меньше чем
- **`>=`** - больше или равно
- **`<=`** - меньше или равно
- **`Определенный`** - имеет значение
- **`Не определено`** - не имеет значения

### Для дат
- **`=`** (Равно) - точная дата
- **`Не равно`** - не равно
- **`>`** - после даты
- **`<`** - до даты
- **`>=`** - после или в дату
- **`<=`** - до или в дату
- **`Определенный`** - имеет значение
- **`Не определено`** - не имеет значения

## Примеры использования

### Строковые условия
```
Тип: Содержит
Значение: "Труба"
Результат: найдет все элементы, содержащие "Труба" в имени

Тип: =  
Значение: "DN100"
Результат: найдет элементы с точным именем "DN100"

Тип: Подстановочный знак
Значение: "DN*"
Результат: найдет элементы, начинающиеся с "DN"

Тип: Определенный
Значение: (пустое)
Результат: найдет элементы, у которых есть имя
```

### Числовые условия
```
Тип: >
Значение: 100
Результат: найдет элементы с числовым значением больше 100

Тип: =
Значение: 50
Результат: найдет элементы со значением равным 50
```

## Рекомендации по использованию

1. **`Содержит`** - самый универсальный для поиска по частичному совпадению
2. **`=`** - для точного поиска по известному значению
3. **`Подстановочный знак`** - для сложных шаблонов с символами подстановки
4. **`Определенный`** - для проверки наличия значения
5. **`Не определено`** - для поиска элементов без значения

## Символы подстановки (Wildcard)

- **`*`** - любое количество символов
- **`?`** - один символ

Примеры:
- `DN*` - все элементы, начинающиеся с "DN"
- `*Труба*` - все элементы, содержащие "Труба"
- `DN???` - элементы с "DN" и тремя любыми символами

## Примечания

- Регистр символов обычно не учитывается
- Пробелы в начале и конце значений автоматически обрезаются
- Для числовых условий значения должны быть числами
- Для дат используйте формат, принятый в вашей системе
